# Choix technologiques – Projet EcoTrack

## Objectif
Ce document présente et justifie les choix technologiques retenus pour le projet EcoTrack afin d’assurer cohérence, maintenabilité et performance.

---

## Backend

### Langage : JavaScript (Node.js)
**Justification :**
- Langage maîtrisé par l’équipe
- Large écosystème
- Très adapté aux API REST

### Framework : Express.js
**Justification :**
- Léger et flexible
- Standard de facto pour les API Node.js
- Facile à intégrer avec des middlewares

---

## Frontend

### Framework : React
**Justification :**
- Composantisation claire
- Forte communauté
- Compatible avec une architecture SPA moderne

---

## Base de données

### SGBD : PostgreSQL
**Justification :**
- Base relationnelle robuste
- Bonne gestion des relations et des contraintes
- Adaptée aux données structurées d’EcoTrack

---

## Conteneurisation & Environnement

### Docker & Docker Compose
**Justification :**
- Environnement reproductible
- Simplifie le développement et le déploiement
- Aligné avec les bonnes pratiques DevOps

---

## Qualité & Outils de développement

### ESLint & Prettier
**Justification :**
- Qualité de code homogène
- Réduction des erreurs
- Facilite le travail en équipe

### Git & GitHub
**Justification :**
- Gestion de versions
- Travail collaboratif par branches
- Intégration CI/CD future

---

## Documentation & Gestion de projet

### Méthodologie : Scrum
**Justification :**
- Travail itératif par sprints
- Meilleure visibilité de l’avancement
- Adapté à un projet académique collaboratif
